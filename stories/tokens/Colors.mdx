# Colors

Tokens live in `app/globals.css` on `:root` (light) and `.dark` (dark). Every color uses the **OKLCH** color space. Tailwind v4's `@theme inline` block exposes each variable as a utility (e.g. `--primary` → `bg-primary`, `text-primary`).

> **Never use raw hex or arbitrary Tailwind palette colors for product surfaces.** Always reference the semantic tokens below. The only exception is the status triad (emerald / amber / red) used by the health badge — see the bottom of this page.

## Adding a new token

1. Add the variable to **both** `:root` and `.dark` in `app/globals.css`.
2. Add the alias inside `@theme inline { --color-<name>: var(--<name>); }`.
3. Use the resulting `bg-<name>` / `text-<name>` utility — never reach into `var(--...)` manually outside the token file.
