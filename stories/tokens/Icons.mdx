# Icons

**One library, one rule:** every icon in the product comes from [`lucide-react`](https://lucide.dev). No SVG sprite, no custom icon component. If a Figma vector doesn't have a clean lucide equivalent, ask before adding it.

## Standard sizes

| Class | Size | Where |
|---|---|---|
| `size-3` | 12 | Avatar status dot |
| `size-3.5` | 14 | Inline badge icons (e.g. `HealthBadge`) |
| `size-4` | 16 | **Default** — inside Button, inputs, sidebar, table cells |
| `size-5` | 20 | Header action icons (theme toggle) |

> `Button` auto-applies `size-4` to any unsized SVG child, so you usually don't need to set a size class on icons inside a button.

## Typed prop pattern

When a component accepts an icon as a prop, type it as `LucideIcon`:

```tsx
import type { LucideIcon } from "lucide-react"

interface StatCardProps {
  icon: LucideIcon
}

function StatCard({ icon: Icon }: StatCardProps) {
  return <Icon className="size-4 text-muted-foreground" />
}
```

## Importing

```tsx
import { Home, FolderOpen, type LucideIcon } from "lucide-react"
```

Pascal-case the icon name exactly as it appears on [lucide.dev](https://lucide.dev). Do not alias unless you need to dodge a name clash.
